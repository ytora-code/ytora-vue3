# 动态表单规则

基于 Naive UI 的动态表单组件，通过 JSON 数组描述表单项，实现“数据驱动页面”的表单渲染方式。

本文档用于定义该组件的最终规则、字段结构、布局行为和使用方式。

## 1 设计目标

该组件遵循以下目标：

1. 24 栅格自动布局
2. 表单项按顺序自动排布
3. 表单项通过宽度和高度控制占位
4. 自定义插槽和自定义渲染作为扩展能力

---

## 2 布局规则

### 2.1 栅格系统

表单容器使用 24 栅格布局。

每个表单项在布局层面具有两个核心属性：

1. `span`：横向占据的栅格数
2. `rowSpan`：纵向占据的行数

布局系统根据表单项数组顺序自动计算每一项的位置。

### 2.2 默认值

表单项布局默认值如下：

1. `span` 默认值为 `24`
2. `rowSpan` 默认值为 `1`

这表示：

- 默认每个表单项独占一整行
- 如果需要一行展示多个表单项，则显式传入不同的 `span`

### 2.3 自动排布规则

表单项位置采用自动排布规则：

1. 表单项按数组顺序依次计算位置
2. 当前行剩余空间足够时，表单项排在当前行
3. 当前行剩余空间不足时，表单项自动换到下一行
4. 换行属于正常布局行为，不视为异常
5. 后续表单项继续基于新的行位置顺序排布

### 2.4 宽高边界

布局边界规则如下：

1. `span` 最大值为 `24`
2. `span <= 0` 时按 `24` 处理
3. `rowSpan <= 0` 时按 `1` 处理
4. `span > 24` 时按 `24` 处理，并在开发环境输出警告

### 2.5 容器尺寸

表单容器支持以下尺寸能力：

1. 可设置最小高度
2. 可设置最大高度
3. 可设置最小宽度
4. 可设置最大宽度
5. 当内容高度超过最大高度或父容器可用高度时，表单内部出现滚动条

### 2.6 坐标能力

动态表单默认不要求使用者传入坐标。

如确有特殊布局需求，可保留以下高级能力：

1. `startPoint`
2. `endPoint`

当同时传入 `startPoint` 和 `endPoint` 时，当前表单项使用显式坐标渲染；否则使用自动排布规则。

该能力属于高级用法，常规业务表单直接使用 `span` 和 `rowSpan` 即可。

## 3 表单项 JSON 结构

表单项由三部分属性组成：

1. 布局属性
2. 通用属性
3. 类型扩展属性

### 3.1 布局属性

用于描述当前表单项在表单中的占位方式。

1. `startPoint`：左上角坐标，可选
2. `endPoint`：右下角坐标，可选
3. `span`：横向占据的栅格数量，可选，默认 `24`
4. `rowSpan`：纵向占据的行数，可选，默认 `1`

说明：

1. 当同时传入 `startPoint` 和 `endPoint` 时，优先使用显式坐标
2. 未传坐标时，组件根据 `span`、`rowSpan` 和顺序自动计算位置

### 3.2 通用属性

通用属性用于定义当前表单项的基础信息和渲染行为。

1. `label`：表单项标题
2. `labelPosition`：标题位置，支持 `left` 或 `top`
3. `labelWidth`：标题宽度
4. `size`：组件大小
5. `placeholder`：占位提示
6. `type`：表单项类型
7. `key`：表单项唯一标识
8. `dataKey`：绑定到表单数据对象中的字段名
9. `dictCode`：字典编码，当 `type` 为 `dict` 时使用
10. `hidden`：是否隐藏当前项
11. `disabled`：是否禁用当前项
12. `defaultValue`：默认值
13. `rules`：当前项校验规则

### 3.3 类型扩展属性

用于透传底层组件属性和事件。

1. `prop`：当前表单项组件的属性对象，渲染时直接透传给底层组件
2. `on`：当前表单项组件的事件对象，渲染时绑定到底层组件

说明：

1. `prop` 用于承载组件特有参数，例如 `clearable`、`options`、`multiple`
2. `on` 用于承载组件特有事件，例如 `change`、`blur`、`focus`

## 4 推荐字段说明

### 4.1 `type`

`type` 表示表单项的组件类型，用于决定底层渲染哪个组件。

例如：

1. `input`
2. `textarea`
3. `select`
4. `dict`
5. `radio`
6. `checkbox`
7. `switch`
8. `date-picker`

### 4.2 `dataKey`

`dataKey` 表示当前表单项绑定到表单数据对象中的字段。

例如：

```ts
{
  type: 'input',
  dataKey: 'userName'
}
```

表示该输入框最终绑定到：

```ts
model.userName
```

### 4.3 `prop`

`prop` 用于透传底层组件属性。

例如：

```ts
prop: {
  clearable: true,
  maxlength: 20
}
```

### 4.4 `on`

`on` 用于透传底层组件事件。

例如：

```ts
on: {
  change: (value) => {
    console.log(value)
  }
}
```

## 5 默认行为

为了降低使用成本，动态表单组件采用以下默认行为：

1. 未传 `span` 时默认为 `24`
2. 未传 `rowSpan` 时默认为 `1`
3. 未传 `labelPosition` 时默认为 `left`
4. 未传 `labelWidth` 时使用表单全局默认值
5. 未传 `size` 时使用表单全局默认值
6. 未传 `placeholder` 时，按 `type + label` 自动生成默认提示
7. 未传 `prop` 时按空对象处理
8. 未传 `on` 时按空对象处理
9. 未传 `hidden` 时默认为 `false`
10. 未传 `disabled` 时默认为 `false`

### 5.1 默认 placeholder 规则

默认 placeholder 按组件类型自动生成：

1. `input` -> `请输入${label}`
2. `textarea` -> `请输入${label}`
3. `select` -> `请选择${label}`
4. `dict` -> `请选择${label}`
5. `radio` -> `请选择${label}`
6. `checkbox` -> `请选择${label}`
7. `date-picker` -> `请选择${label}`

如果显式传入 `placeholder`，则优先使用用户传入的值。

## 6 完整表单项示例

```ts
{
  span: 12,
  rowSpan: 1,
  label: '用户名称',
  labelPosition: 'left',
  labelWidth: 120,
  size: 'medium',
  placeholder: '请输入用户名',
  type: 'input',
  key: 'userName',
  dataKey: 'userName',
  hidden: false,
  disabled: false,
  defaultValue: '',
  rules: [
    {
      required: true,
      message: '请输入用户名',
      trigger: 'blur',
    },
  ],
  prop: {
    clearable: true,
    maxlength: 20,
  },
  on: {
    blur: () => {},
  },
}
```

## 7 表单项数组

完整表单通过表单项数组描述。

示例：

```ts
const schemas = [
  {
    span: 12,
    label: '用户名称',
    type: 'input',
    dataKey: 'userName',
    prop: {
      clearable: true,
    },
  },
  {
    span: 12,
    label: '状态',
    type: 'dict',
    dataKey: 'status',
    dictCode: 'sys_normal_disable',
    prop: {
      clearable: true,
    },
  },
]
```

表单组件接收到该数组后，即可完成整个表单的渲染。

## 8 表单组件 API

表单组件本身也具有一组固定能力。

### 8.1 Props

1. `modelValue`：表单绑定的数据对象
2. `schemas`：表单项配置数组
3. `rules`：表单级校验规则
4. `labelWidth`：全局标题宽度
5. `labelPosition`：全局标题位置
6. `size`：全局组件大小
7. `minHeight`：表单最小高度
8. `maxHeight`：表单最大高度
9. `minWidth`：表单最小宽度
10. `maxWidth`：表单最大宽度
11. `loading`：提交中状态
12. `disabled`：整个表单是否禁用
13. `showFeedback`：是否显示字段校验反馈区域，默认 `true`
14. `showActionRow`：是否显示底部操作栏
15. `submitText`：提交按钮文案
16. `resetText`：重置按钮文案
17. `submitIcon`：提交按钮图标字符串
18. `resetIcon`：重置按钮图标字符串
19. `actionPlacement`：操作区位置，支持 `bottom-left`、`bottom-right`、`inline`

说明：

1. `submitIcon` 和 `resetIcon` 直接传图标字符串即可，组件内部统一通过图标工具渲染
2. `bottom-left` 表示操作区单独起一行并左对齐
3. `bottom-right` 表示操作区单独起一行并右对齐
4. `inline` 表示操作区作为最后一个栅格项追加到表单尾部，当前行空间不足时自动换到下一行

### 8.2 Events

1. `update:modelValue`：表单值更新事件
2. `submit`：提交事件
3. `reset`：重置事件
4. `field-change`：字段值变化事件

### 8.3 Expose

组件对外暴露以下方法：

1. `validate`：校验整个表单
2. `reset`：重置整个表单
3. `setFieldsValue`：批量设置字段值
4. `getFieldsValue`：获取当前表单值
5. `scrollToField`：滚动到指定字段

## 9 校验规则

动态表单使用字段规则和表单规则组合校验。

校验规则来源：

1. 表单项自身的 `rules`
2. 表单组件传入的全局 `rules`

两者合并后交给 Naive UI 表单能力执行校验。

### 9.1 使用原则

1. 单字段规则优先写在表单项的 `rules` 中
2. 跨字段规则写在表单级 `rules` 中

## 10 插槽规则

动态表单支持使用插槽替换默认表单项渲染。

### 10.1 表单级插槽

支持以下表单级插槽：

1. `header`
2. `footer`
3. `actions`

### 10.2 字段插槽

字段插槽命名规则为：

```vue
#${key}
```

例如：

```vue
<template #userName="{ model, schema }">...</template>
```

当字段插槽存在，且其名称与表单项 `key` 对应时，该字段使用插槽内容渲染。

例如：

```ts
{
  key: 'custom',
  type: 'input',
  dataKey: 'customText'
}
```

对应插槽：

```vue
<template #custom="{ model }">
  <n-input v-model:value="model.customText as string" />
</template>
```

此时插槽内容会替换该表单项的默认渲染。

### 10.3 渲染优先级

渲染优先级如下：

1. 自定义 `render`
2. 字段插槽
3. 默认组件渲染

如果组件实现了 `render` 能力，则按以上优先级执行。

## 11 使用示例

### 11.1 基础示例

```vue
<script setup lang="ts">
import { ref } from 'vue'

const formModel = ref({
  userName: '',
  status: null,
})

const schemas = [
  {
    span: 12,
    label: '用户名称',
    type: 'input',
    dataKey: 'userName',
    prop: {
      clearable: true,
    },
  },
  {
    span: 12,
    label: '状态',
    type: 'dict',
    dataKey: 'status',
    dictCode: 'sys_normal_disable',
    prop: {
      clearable: true,
    },
  },
]

const handleSubmit = () => {}

const handleReset = () => {}
</script>

<template>
  <DynamicForm
    v-model:modelValue="formModel"
    :schemas="schemas"
    action-placement="inline"
    submit-text="搜索"
    reset-text="重置"
    submit-icon="i-lucide-search"
    reset-icon="i-lucide-rotate-ccw"
    @submit="handleSubmit"
    @reset="handleReset"
  />
</template>
```

### 11.2 自定义字段插槽

```vue
<DynamicForm v-model:modelValue="formModel" :schemas="schemas">
  <template #field-userName="{ model }">
    <n-input v-model:value="model.userName" placeholder="请输入用户名" />
  </template>
</DynamicForm>
```
